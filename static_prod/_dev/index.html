FAQ

How does using ugnis look like? You use our visual editor to create the definition of your application (logged in users can save one definition and come back to it later on). Once your app is ready, you can export a minified javascript code and put it in your website.

Can Ugnis create an app for Windows, Linux and Mac? Yes, when exporting you can select to export as a wrapped Electron app.

Can Ugnis create and iOS or Android app? Not yet, though in the future a slightly changed editor with more specialized components for every OS will be able to export to iOS and Android

Should you use ugnis?

For React, Redux, Angular or Elm programmers

  • Full framework with view, state management, styling, routing and http requests
  • Visually shows how parts of the app are connected
  • Type safe without getting in your way
  • Less boilerplate, faster development speed
  • No build tools required
  • Can be introduced in small steps to an existing app
  • Faster and much smaller than any other framework
  • Build for scale - large apps are easy to understand and change
  • Early in development - rapid UI changes, lacking examples
  • Some very specific styles and behaviours are not yet possible, but you can always add CSS classes and external javascript
  • Support for multiple people working on the same project is in an early phase
  • To implement real time collaboration and git-like history your code has to be on Ugnis servers

For people without programming experience

  • As powerful as a programming language - create any app you can think of
  • Simple - become productive in a few hours
  • Guaranteed safety - no fear of making a mistake
  • Enables everyone on the team - programmers, designers, managers to make changes
  • One of the first real development environment to work on mobile
  • Does not yet provide data storage for your apps - but there are plenty backend solution to chose from like Firebase

For non-JavaScript programmers

Just use ugnis. We don't recommend going to the JavaScript world of undefined with bazillion different frameworks and bundlers. We've been there, 'tis a silly place.
